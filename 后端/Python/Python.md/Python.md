# 虚拟环境

虚拟环境的作用在项目开发上非常有用，目的是为项目创建一个运行的虚拟环境，项目之间依赖的软件包互相不干扰，当你的朋友想要运行项目时，也不会出现依赖的软件包版本问题而导致项目无法启动。

## 创建虚拟环境

创建虚拟环境命令如下:

```bash
python -m venv .venv
```

> [!NOTE]
>
> .venv: 是创建的虚拟环境名称，行业上推荐写死为`.venv`或`venv`。如果觉得这个名称不喜欢，可以自定义其他名称，但是推荐向行业看齐。

## 激活虚拟环境

激活虚拟环境命令如下：

```bash
# Windows
.venv\Scripts\activate

# Mac
source .venv/bin/activate
```

> [!NOTE]
>
> 激活虚拟环境操作，要进入到项目根目录再执行，`.venv`其实就是创建的虚拟环境目录，上面的命令实际就是进入这个目录执行`activate`程序。

## 退出虚拟环境

退出虚拟环境命令如下：

```bash
deactivate
```
